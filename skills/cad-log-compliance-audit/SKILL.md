# CAD Log Compliance Audit Framework

## Purpose
Passive audit layer. When user provides or describes CAD (Computer-Aided Dispatch) logs or dispatch records, automatically assess against California POST Dispatcher standards, APCO/NENA operational standards, California Government Code emergency response requirements, and documented best practices for dispatch operations.

## Governing Standards

### California POST — Public Safety Dispatcher Training
- POST Basic Dispatcher Course (120 hours minimum)
- 14 Dispatcher Learning Domains
- POST Dispatcher Continuing Professional Training
- Reference: post.ca.gov

- DLD 9 — Critical Incidents

- DLD 12 — Child/Elder and Dependent Abuse
- DLD 13 — Missing Persons
- DLD 14 — Resources/Referral Services

### APCO Standards (Association of Public-Safety Communications Officials)
- APCO ANS 1.101.3 — Core Competencies for Public-Safety Communications
- APCO ANS 3.103.1 — Minimum Training Standards for Telecommunicators
- APCO P33 — Minimum Training Standards for Telecommunicators (Project 33)

- GC 53100-53109 — Local emergency services

### California Health & Safety Code

## Audit Triggers

Activate when user provides or describes:
- CAD log or dispatch printout
- Call-for-service record
- 911 call record or transcript
- Dispatch decision or prioritization
- Response time data
- Radio communication log
- MDT (Mobile Data Terminal) communication
- Resource allocation decision
- Dispatch-to-arrival timeline
- Any communication between dispatch and field units

## CAD Log Required Elements

### Timestamp Fields (All Must Be Present)
1. **Call Received Time** — When 911/admin call was answered
2. **Call Entered Time** — When call was entered into CAD system
3. **Unit Dispatched Time** — When officer/unit was assigned
4. **Unit En Route Time** — When unit acknowledged and began response
5. **Unit On Scene Time** — When unit arrived at location
6. **Unit Cleared Time** — When unit completed the call

**Audit Check**: Are all six timestamps present? Any gaps represent documentation failures. The delta between timestamps reveals response time compliance and potential issues.

### Call Classification Fields
7. **Call Type/Nature Code** — Priority classification of incident
8. **Priority Level** — Typically P1 (emergency/in-progress) through P4 (administrative)
9. **Location** — Address or intersection
10. **Reporting Party Information** — Who called, callback number
11. **Call Summary/Narrative** — Dispatcher's description of reported incident
12. **Disposition Code** — How the call was resolved

### Unit Assignment Fields
13. **Primary Unit** — First unit assigned
14. **Assisting Unit(s)** — Additional units
15. **Supervisor Notification** — If required by call type
16. **Specialized Unit Request** — Detectives, CSI, K9, SART, etc.

### Communication Fields
17. **Dispatch-to-Unit Communications** — Key broadcasts
18. **Unit-to-Dispatch Updates** — Status changes, additional information
19. **BOLO/ATL Broadcasts** — Be On the Lookout / Attempt to Locate
20. **Mutual Aid Requests** — If other agencies requested

## Audit Framework

### Phase 1: Completeness Assessment
For each CAD log entry, verify:
1. Are all six timestamp fields populated?
2. Is call type/priority classification present and appropriate for described incident?
3. Is location specific enough for response (full address vs. vague area)?
4. Is reporting party information captured?
5. Is disposition code present and consistent with outcome?

### Phase 2: Timeline Analysis
Calculate and assess:
1. **Call Processing Time**: Call Received → Unit Dispatched
   - Standard: Priority 1 calls should be dispatched within 60-90 seconds
   - Extended processing times may indicate staffing issues or improper prioritization
2. **Response Time**: Unit Dispatched → Unit On Scene
   - Compare against agency's published response time standards
   - Note: No statewide mandatory response time standard, but agencies typically set their own
3. **Total Response Time**: Call Received → Unit On Scene
   - This is the metric most relevant to the person who called
4. **On-Scene Time**: Unit On Scene → Unit Cleared
   - Unusually short on-scene times may indicate inadequate investigation
   - Unusually long times without documented activity may indicate other issues

### Phase 3: Prioritization Assessment
Evaluate whether call was properly prioritized:
- **Priority 1 (Emergency/In-Progress)**: Crimes in progress, life-threatening situations, officer needs help
- **Priority 2 (Urgent)**: Just-occurred crimes, DV reports, disturbances
- **Priority 3 (Routine)**: Cold reports, property crimes not in progress
- **Priority 4 (Administrative)**: Follow-up, paper service, information

**Audit Check**: Was a DV call classified as Priority 1 or 2? Was a crime in progress classified as Priority 1? Downgrading priority can result in delayed response and exacerbated harm.

### Phase 4: Cross-Reference with Police Report
When both CAD log and police report are available:
1. Do CAD timestamps match report timestamps?
2. Does CAD call narrative match report narrative?
3. Are all units shown in CAD also identified in the report?
4. Does CAD disposition match report disposition?
5. Were additional units or specialized resources requested per CAD but not documented in report?

**Discrepancies between CAD and police report are significant audit findings.** CAD logs are generated in real-time by dispatch and are harder to alter retroactively than narrative reports.

### Phase 5: Statutory Compliance
Assess against specific requirements:
- For DV calls: Was call properly prioritized? Were mandatory arrest provisions communicated to responding unit?
- For missing persons: Was report accepted immediately per PC 14205?
- For child abuse reports: Were cross-reporting procedures initiated from dispatch level?
- For mental health calls: Were crisis intervention resources dispatched or offered?
- For elder/dependent adult abuse: Were APS notification procedures initiated?

## Output Protocol

When CAD log issue detected, inject:
```
[CAD LOG AUDIT FLAG]
Agency: [Department/dispatch center]
CAD Entry: [Case/event number and date]
Element: [Which required field or standard]
Standard: [POST DLD / APCO / statutory requirement]
Finding: [What the log shows vs. what was required]
Timeline Analysis:
  - Call Processing: [X minutes — within/exceeds standard]
  - Response Time: [X minutes — within/exceeds standard]
  - On-Scene Duration: [X minutes — adequate/inadequate for call type]
Classification: [Timestamp Gap / Priority Misclassification / Documentation Deficiency / Report Discrepancy / Statutory Non-Compliance]
Severity: [Critical / Major / Minor]
Cross-Reference: [If police report also available — note discrepancies]
Remedies: [Agency complaint / Grand jury referral / Civil claim / Pattern evidence for federal claim]
```

## CAD-to-Report Discrepancy Protocol

When both CAD log and police report are available for the same incident:
```
[CAD-REPORT DISCREPANCY]
Incident: [Case number and date]
CAD Shows: [What the dispatch record documents]
Report Shows: [What the police report documents]
Discrepancy: [Specific conflict or omission]
Significance: [Why this matters — e.g., "CAD shows 47-minute response time; report omits arrival time entirely"]
Evidentiary Note: [CAD logs are contemporaneous real-time records; discrepancies favor CAD log reliability]
```
